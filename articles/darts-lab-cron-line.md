## Cronジョブ — アプリの目覚まし時計

**Cronジョブ** は「決まった時間に自動で処理を実行する」仕組みだ。スマホの目覚まし時計と同じで、毎朝決まった時間に起動して仕事をしてくれる。

darts Lab では毎日午前10時（JST）に、登録ユーザーのダーツスタッツを自動取得し、分析し、LINE で通知している。

## 9ステップの日次バッチ処理

毎朝の自動処理は、以下の9ステップで動いている。

```
① LINE連携済みユーザーを取得
    ↓
② 暗号化されたDARTSLIVE認証情報を復号
    ↓
③ Puppeteerでブラウザを起動
    ↓
④ DARTSLIVEサイトにログインしてスタッツを取得
    ↓
⑤ 前回データとの差分を検出
    ↓
⑥ Firestoreにスタッツを保存
    ↓
⑦ XPを付与（14のルールに基づく）
    ↓
⑧ 目標の達成判定
    ↓
⑨ LINEで結果を通知
```

## Webスクレイピング — ロボットがWebページを読む

**スクレイピング** とは、プログラムが自動でWebサイトにアクセスしてデータを取得する技術だ。

なぜ必要か？DARTSLIVE は公式のAPIを提供していない。つまり「プログラムからデータをください」と正規のルートでお願いする方法がない。そこで **Puppeteer** というツールを使って、人間がブラウザで操作するのと同じ動作を自動化している。

**Puppeteer** は「画面のないChrome」（ヘッドレスブラウザ）を操作するツールだ。

```
Puppeteerがやっていること:
1. Chromeを起動
2. DARTSLIVEのログインページを開く
3. メールアドレスとパスワードを入力
4. ログインボタンをクリック
5. スタッツページに移動
6. 表示されたデータを読み取る
7. Chromeを閉じる
```

この一連の操作が30〜60秒かかる。だからこそ、毎回ユーザーを待たせるのではなく、Cronジョブで事前に取得してキャッシュしておく設計にしている。

## LINE Bot — LINEの中にロボットを配置

**LINE Messaging API** を使うと、LINEの中にBot（自動応答プログラム）を置ける。

darts Lab のLINE Botは以下の機能を持つ。

| 機能 | 説明 |
|------|------|
| アカウント連携 | 6桁コードでWebアカウントとLINEを紐づけ |
| 日次通知 | 毎朝のスタッツ変動をLINEで通知 |
| 即時取得 | 練習後にBotからスタッツを即時取得 |
| メモ | 練習メモをLINEから記録 |

### Webhookで双方向通信

LINE Botも **Webhook** の仕組みで動いている。ユーザーがLINEでメッセージを送ると、LINEのサーバーからdarts Lab のサーバーにリクエストが届く。サーバーが内容を処理して、LINE APIを通じて返信する。

### なりすまし防止

LINEのWebhookも、Stripeと同様に署名検証が必要だ。

```
HMAC-SHA256 + crypto.timingSafeEqual()
```

**HMAC-SHA256** は「このリクエストが本当にLINEから来たか」を確認する仕組み。`timingSafeEqual()` は **タイミング攻撃** を防ぐための比較関数だ（通常の文字列比較は処理時間から情報が漏れる可能性がある）。

## 暗号化 — ユーザーの認証情報を守る

DARTSLIVEの自動データ取得には、ユーザーのログイン情報が必要だ。これを平文（そのまま読める状態）で保存するのは論外。**AES-256-GCM** で暗号化している。

| 要素 | 役割 |
|------|------|
| AES-256 | 暗号化アルゴリズム（軍事レベルの強度） |
| GCM | 改ざん検出モード |
| IV（初期化ベクトル） | 毎回ランダム生成。同じパスワードでも異なる暗号文になる |
| 認証タグ | データが途中で改ざんされていないか検証 |
| 暗号鍵 | 環境変数に保管（ソースコードには含めない） |

## まとめ

- Cronジョブで毎朝自動的にデータを取得
- Puppeteerで「画面のないChrome」を操作してスクレイピング
- LINE Botで通知・即時取得・メモを実現
- Webhook + 署名検証でセキュリティを担保
- AES-256-GCMでユーザーの認証情報を暗号化
