## 要件定義とは？

家を建てる前に設計図を書くように、アプリを作る前に「何を作るか」を決める作業が **要件定義** だ。

「とりあえずコードを書き始める」のと「まず何を作るか整理する」のでは、完成するアプリの品質が大きく変わる。個人開発で最も危険なのは **スコープの膨張**（あれもこれもと機能が増えて、永遠に完成しない）だ。

## ペルソナ — 架空のユーザー像を作る

**ペルソナ** とは「このアプリを使う架空のユーザー像」のこと。「誰のためのアプリか？」を具体的にする作業だ。

darts Lab では3人のペルソナを定義した。

| ペルソナ | 属性 | 主なニーズ |
|---------|------|----------|
| Aさん（初心者） | ダーツ歴3ヶ月, Rt.3 | 他の人のセッティングを参考にしたい |
| Bさん（中級者） | ダーツ歴2年, Rt.8 | バレル比較、スタッツ分析 |
| Cさん（上級者） | ダーツ歴5年, Rt.14 | 情報発信、記事執筆 |

自分自身はBさんとCさんの中間だが、あえて **Aさん（初心者）を含めた** のがポイント。自分のためだけに作ると視野が狭くなる。「ダーツを始めたばかりの人が、他のプレイヤーのセッティングを見て参考にする」シーンを想定したことで、セッティングの公開・閲覧機能の優先度が上がった。

## ユーザーストーリー — 「〇〇したい、なぜなら△△だから」

ペルソナごとに **ユーザーストーリー** を書き出す。

```
Bさんは、今使っているバレルに近い重量・形状の別ブランドを探したい
  → バレルDB + スペック検索 + レコメンド機能

Bさんは、先月と今月のスタッツ推移を比較して成長を確認したい
  → スタッツ期間比較 + 移動平均トレンド
```

「したいこと」から機能を導くことで、「使われない機能」を作るリスクを減らせる。

## MVP — 最小限で価値を届ける

ユーザーストーリーから機能候補を出すと、40以上の候補が出てくる。全部を一度に作るのは無理なので、優先度を3段階に分ける。

```
Priority 1（MVP）: これがないとアプリとして成立しない
  - ユーザー認証
  - セッティング登録・一覧
  - バレルDB・検索
  - スタッツ記録

Priority 2（初期リリース後）: ユーザー価値が高い
  - DARTSLIVE連携（自動スタッツ取得）
  - スキル分析
  - ショップ管理

Priority 3（成長フェーズ）: あると差別化できる
  - Stripe決済（PROプラン）
  - LINE通知連携
  - XP/ランクシステム
```

**MVP**（Minimum Viable Product）とは「最小限の価値を持つ製品」。判断基準はシンプルで、**「これがないと毎日使わないか？」** で切った。自分自身がユーザーだから、この判断はブレなかった。

## 機能一覧と非機能要件

最終的に darts Lab では **23の機能定義**（F-001〜F-023）と **4つの非機能要件** を整理した。

### 機能要件の例

| ID | 機能 | 概要 |
|----|------|------|
| F-001 | ユーザー認証 | メール/Google でログイン |
| F-005 | バレルDB | 7,000件以上のバレルを検索 |
| F-010 | DARTSLIVE連携 | スタッツを自動取得 |
| F-015 | XPシステム | プレイに応じてXPを付与 |

### 非機能要件

機能以外に「アプリとしてどうあるべきか」も定義する。

| カテゴリ | 要件 |
|---------|------|
| パフォーマンス | ダッシュボード表示 1秒以内 |
| セキュリティ | 認証情報の暗号化、ロール昇格防止 |
| 可用性 | Vercel + Firebase で 99.9% 稼働 |
| テスト | 重要ロジックのユニットテスト必須 |

## AIと要件定義する

darts Lab の要件定義は **Claude Code** と対話しながら作成した。

AIが得意なこと:
- **構造化**: 散らばった不満やアイデアを、ペルソナ・ユーザーストーリー・機能一覧に整理
- **網羅性**: 非機能要件のチェックリストを体系的に洗い出し

人間がやるべきこと:
- **ドメイン知識**: 「DARTSLIVEのフライト階級はC〜SAまである」はAIにはわからない
- **優先度判断**: 「これがないと毎日使わない」の判断はユーザーである自分にしかできない
- **プロンプト設計**: スコープを絞らないとAIが違う方向に突き進む

## まとめ

- 要件定義 = 「何を作るか」を決める技術
- ペルソナで「誰のため」を明確にする
- ユーザーストーリーで「したいこと」から機能を導く
- MVP でスコープを絞り、最小限から始める
- AIは構造化が得意だが、ドメイン知識と優先度判断は人間の仕事
