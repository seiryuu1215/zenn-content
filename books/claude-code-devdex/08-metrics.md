## テストカバレッジの推移

```
v0 MVP:    403件（CRUD API + コアUI）
v1:        496件（マスターデータ + 関連用語）
v2:       1,025件（診断 + スキルシート + 公開プロフィール）
v3:       1,316件（組織機能 + Feature Gate + Admin）
公開準備: 1,407件（LP + オンボーディング + 診断デモグラフィック）
機能追加: 1,813件（ダッシュボード改善 + バグ修正 + UI改善）
品質強化: 2,100+件（Pro限定機能 + AI機能拡充 + テスト網羅性向上）
```

v0からv1にかけてはテスト数の増加が緩やかだが、v2からv3で急増している。権限・ロールの組み合わせテスト（3個人ロール x 4組織ロール x 11機能ゲート）がテスト数を押し上げた。公開準備以降も、Pro限定機能やAI機能の拡充に伴ってテストは継続的に増加し、最終的に2,100+件に到達した。

## 1日あたりの生産性

v0のMVP（14 issue）を実働3〜4日で完了した。1日あたり3〜5個のPRがマージされるペースだ。1 issueあたりの平均実装時間は1〜2時間。PM Agentのタスク分解で「1 issue = 1〜2時間」ルールを設定した効果がここに表れている。

サブエージェントフロー1周（PM → 実装 → テスト → レビュー）は、小さなissueなら30分、大きなissueでも2時間以内に完了した。

## AI駆動 vs 手動コーディングの比較

手動でこの規模のアプリを作った場合の工数を見積もると、少なくとも3〜4倍の時間がかかるだろう。特にテストの2,100+件を手動で書く作業と、全APIの認証チェック・バリデーション・エラーハンドリングを統一的に実装する作業は、AI無しでは数週間かかる。

543+件のPRがすべて「概要・変更内容・設計判断・確認方法」のテンプレートに沿い、コミットメッセージも統一フォーマットで書かれている。この一貫性は人間のチーム開発では達成困難だ。
