```
v0 MVP:    403件（CRUD API + コアUI）
v1:        496件（マスターデータ + 関連用語）

v3:       1,316件（組織機能 + Feature Gate + Admin）
公開準備: 1,407件（LP + オンボーディング + 診断デモグラフィック）

品質強化: 2,350件（Pro限定機能 + AI機能拡充 + テスト網羅性向上）
```

v0からv1にかけてはテスト数の増加が緩やかだが、v2からv3で急増している。権限・ロールの組み合わせテスト（3個人ロール x 4組織ロール x 11機能ゲート）がテスト数を押し上げた。公開準備以降も、Pro限定機能やAI機能の拡充に伴ってテストは継続的に増加し、最終的に2,350件に到達した。

## 1日あたりの作業時間

6暦日・実質5日間の開発を、1日ごとに振り返る。

| 日 | 稼働時間 | コミット | やったこと | 備考 |
|----|---------|---------|----------|------|
| Day 1 (火) | ~15h | 86 | v0 MVP（用語CRUD・認証・ダッシュボード） | 08:14〜23:13 |
| Day 2 (水) | ~15h | 61 | v1（AI機能・診断・スキルシート）| 深夜0:50スタート |
| Day 3 (木) | ~16h | 52 | v2（組織・公開プロフィール・ローンチ準備） | 05:49〜22:10 |
| Day 4 (金) | ~10h | 60 | LP・SEO・法務・Stripe課金 | **ダーツプロイベントで日中不在** |
| Day 5 (土) | ~6h | 18 | Pro限定機能（面談シミュレーター・学習プラン） | **ダーツプロイベントで日中不在** |
| Day 6 (日) | ~14h | 41 | セキュリティ修正・テスト強化・Zenn・ポートフォリオ | コード凍結日 |

**合計: 約76時間・334コミット**

Day 1〜3は1日15時間超の集中開発。Day 4-5はダーツのプロイベントに朝から夜まで参加しながら、会場の隙間時間にスマホからClaude Codeへ指示を出して開発を進めた。イベント中にSlackでメモを残し、帰宅後にClaude Codeに渡す。人間が判断だけして、実装はAIに任せる -- AI駆動開発ならではの「ながら開発」だ。Day 4は10時間、Day 5は6時間と稼働が落ちているが、コミット数はDay 4が60件と高水準を維持している。サブエージェント体制のおかげで、短い指示でも実装・テスト・レビューの一連のフローが自律的に回った結果だ。

## 1日あたりの生産性

v0のMVP（14 issue）を実働3〜4日で完了した。1日あたり3〜5個のPRがマージされるペースだ。1 issueあたりの平均実装時間は1〜2時間。PM Agentのタスク分解で「1 issue = 1〜2時間」ルールを設定した効果がここに表れている。

サブエージェントフロー1周（PM → 実装 → テスト → レビュー）は、小さなissueなら30分、大きなissueでも2時間以内に完了した。

## AI駆動 vs 手動コーディングの比較

手動でこの規模のアプリを作った場合の工数を見積もると、少なくとも3〜4倍の時間がかかるだろう。特にテストの2,350件を手動で書く作業と、全APIの認証チェック・バリデーション・エラーハンドリングを統一的に実装する作業は、AI無しでは数週間かかる。

637+件のPRがすべて「概要・変更内容・設計判断・確認方法」のテンプレートに沿い、コミットメッセージも統一フォーマットで書かれている。この一貫性は人間のチーム開発では達成困難だ。
