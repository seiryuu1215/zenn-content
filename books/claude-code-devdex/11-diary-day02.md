## この日のゴール

v1の3本柱「AI概要補完」「エンジニアタイプ診断」「スキルシート」を1日で実装する。

## 朝: AI概要補完（#82〜#100）

用語を登録したとき、概要欄を手で書くのが面倒。でも全自動にすると「自分の言葉で理解する」というコンセプトが崩れる。

落とし所は **「概要だけAIが下書き、経験や理解度は必ず手入力」**。Anthropic Claude APIを叩いて、用語名から概要テキストを生成する。ユーザーはそれを編集してもいいし、そのまま使ってもいい。でも `my_experience`（自分の経験）と `interview_summary`（面談で話すこと）は絶対に手入力。

AI概要補完の利用回数はFree: 10回/日に制限した（後にFree: 5回/月に変更）。

## 昼: エンジニアタイプ診断（#101〜#145）

DevDexの集客エンジンになる機能。**登録不要・64問・16タイプ**。

4軸（動機の源泉 / リスクの取り方 / 思考の方向 / 関係の使い方）× 2極 = 16タイプ。MBTIのようなフレームワークだが、エンジニアの仕事スタイルに特化している。

16タイプの設計で一番悩んだのは **タイプ名**。「設計者」「探究者」「職人」「火付け役」——日本語で直感的に分かり、かつ自分のタイプを「見せたくなる」名前にした。

診断結果の保存はlocalStorageに一時保存し、ログイン後にDBに移行する方式。未ログインでも診断できることが集客の生命線だから。

fal.ai Flux Proで16タイプ分のキャラクターアイコンを生成。プロンプトの調整に30分くらい使った。

## 夕方: スキルシート（#146〜#170）

業務経歴と個人開発のプロジェクトを登録し、PDF/Excelでエクスポートする機能。フリーランスが案件に応募するときの技術経歴書を、DevDexのデータから自動生成する。

PDFはreact-pdfで組み立て、ExcelはExcelJSで出力。セル結合やスタイリングで苦戦したが、Implement Agentに「参照フォーマットに合わせて」と指示したら、ほぼ一発で出た。

## 夜: 256パターンプロフィールデータ + ドキュメント整備（#171〜#181）

16タイプ × 16タイプ = 256通りの組み合わせデータを生成。これは後のv2で「統合レポート」として花開く。

要件定義書と基本設計書も作成。PM Agentに「docs/ 以下に設計書を出して」と指示。人間が書くより網羅性が高い設計書が出てきた。
