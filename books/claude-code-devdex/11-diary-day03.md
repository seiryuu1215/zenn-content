## この日のゴール

「使えるもの」から「売れるもの」へ。収益化基盤、ランディングページ、法務対応を一気にやる。

## 朝: Stripe連携 + Feature Gate（#228〜#245）

Free / Pro の機能ゲートを実装。`FeatureGate` コンポーネントと `canAccessFeature()` ユーティリティで、ロールに応じた機能の出し分けを一元管理する。

Stripeのチェックアウト・Webhook・カスタマーポータルを接続。Webhookの署名検証、subscription.updated/deletedの同期処理、100%OFFクーポンの自動解約フラグ——決済周りはバグが許されないので、Review Agentに重点的にチェックさせた。

**判断: 「データを入れるのは無料、外に出すのは有料」**。用語登録・診断・ダッシュボードはFree。PDF/Excel出力・公開プロフィール・詳細分析はPro。使い込んでデータが溜まるほどProの価値が上がる設計。

## 昼: ランディングページ + SEO（#246〜#261）

非エンジニアが来ても「何ができるか」がすぐ分かるLPを作った。ウェルカムモーダル → サンプルデータ投入 → 3ステップオンボーディングの導線も設計。

SEO対策としてOGPメタデータ、robots.txt、sitemap.ts、特定商取引法に基づく表記ページを追加。個人開発でもここは手を抜けない。

## 午後: 診断の強化（#249〜#278）

診断に属性アンケート（年齢・雇用形態・経験年数・主要ドメイン）を追加。これは後でデモグラフィック分析に使う。

診断の軸数を **4軸20問 → 4軸32問** にリプレイス。精度を上げるためだが、「長すぎて離脱しないか」は心配だった。最終的にDay 5で64問に拡張する。

年額プランの料金テーブルも追加。月額500円、6ヶ月2,400円（おすすめ）、年額3,000円。

## 夕方: E2Eテスト + セキュリティ（#266〜#296）

Playwrightの基盤構築。全ロール（admin/pro/user）のstorageStateを自動生成するスクリプトを書いて、E2Eテストシナリオ D-8〜D-14 を追加。

iOSレスポンシブの崩れも修正。モバイルファーストを意識していたつもりだが、実機で見ると細かいズレがある。

## この日の失敗

- **Expoに手を出してしまった（2回目）**。Expo用のログイン画面・ダッシュボード・用語一覧を実装したが、Web版の優先度が高いと判断して以降は凍結。2日連続で同じ過ちを繰り返した。
