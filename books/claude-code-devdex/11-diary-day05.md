## 「完璧」を求めると永遠に出せない

Day 5までで機能はほぼ揃った。でも「あれも直したい」「この機能も欲しい」が次々出てくる。面談シミュレーターのエラーメッセージを改善したい。SNSシェアの文言をもっとキャッチーにしたい。スキルシートのPDF出力を一旦非表示にしたい（品質が不十分だから）。

ここで覚悟を決めた。**「今日で凍結する」**。

完璧なプロダクトは存在しない。出さなければフィードバックも来ない。フィードバックがなければ改善もできない。

## 凍結の反省点

凍結エビデンスに自分で7項目の反省点を記載した:

1. テスト初日にゼロだった → 最初からTDDで進めるべき
2. Expoに2回手を出した → MVP範囲を厳守すべき
3. UIの調整に時間をかけすぎた → 機能優先
4. レビューを一部飛ばした → 品質ゲートは不可欠
5. ドキュメントが後手に回った → CLAUDE.mdの更新を常に最新に
6. E2Eテストが不十分 → 基盤はあるがシナリオが少ない
7. パフォーマンス計測をしていない → Lighthouseスコアの定期計測

**失敗を記録することに価値がある**。次のプロジェクト（Darts Labの改善）でこの反省をそのまま活かせた。
