### 招待URL全パターン検証（#845）

招待URLの期限切れ動作を**全10パターン**洗い出して修正:

1. 未ログイン + 無効コード → エラー表示（以前は「Pro無料！」と表示されていた）
2. 期限切れ当日の通知 → `pro_expired` 通知タイプを追加
3. 期限切れ後の再招待 → `referred_by` を自動リセットして再適用可能に

**「使われない機能はバグが埋まっている」**。招待URLは自分しか使わないからテストが甘かった。

## Day 13 (月): 集客基盤 + ポートフォリオ自動化

### シェアトラッキング（#859）

シェアボタンのクリックイベントをDBに記録する仕組みを追加。`share_events` テーブルで platform（X/LINE/Instagram/コピー）× context（診断/対人/統合/プロフィール）× typeCode を記録。

fire-and-forget（UIをブロックしない非同期送信）で実装。ゲストユーザーも記録可能。

**「計測できないものは改善できない」**。シェア率が分かれば、OGP画像やシェアテキストの改善効果を数字で判断できる。

### 対人診断CTAバナー（#860）

エンジニア診断結果ページに「対人スタイル診断も受けよう → 256通りの統合レポートが解放」のCTAバナーを追加。グラデーション背景で目立たせ、未診断/診断済みで表示を切り替え。

### ポートフォリオ自動更新システム

`npm run update-portfolio` 一発で:
1. Playwrightで本番サイトのスクリーンショット8枚を自動取得
2. gitログからテスト数・PR数・行数等の統計を自動集計
3. ポートフォリオのWorksSectionを自動更新
4. Zenn書籍のタイトル・集計テーブルも自動更新

新サービスを追加しても、設定ファイルに定義を追加するだけで同じ仕組みが使える。

### 管理画面リンク（#858）

adminユーザーのヘッダーに管理画面への導線を追加。盾アイコンでサッと管理画面に行ける。
